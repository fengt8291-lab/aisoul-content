## 二、文案结构

### 2.1 标题规范

**爆款标题公式（至少满足2条）：**

| 公式 | 示例 |
|------|------|
| 数字开头 | "5个AI同时回答"、"3个方法" |
| 疑问句 | "你们有没有被AI说中的经历？" |
| 情绪词+emoji | "让我愣住了"、"让我沉默了很久"😱 |
| 反常识/揭秘感 | "不是找到的，是养成的" |

**禁忌：** 太长（超过20字）、太像广告、太学术

### 2.2 黄金开头（前3句）

**原则：让用户觉得"跟我有关"**

```
[场景/行为] → [具体细节] → [AI的回答/结果] → [情绪反应]😱
```

**反面案例（AI味）：**
> "今天我用AI做了一个灵魂测试，测试结果显示我是探索者型。"

**正确案例（有人味）：**
> "我把闺蜜头像发给AI看。它说：她最近很想谈恋爱，但不想主动。我把这段话发给她，她秒读已读不回。"

### 2.3 正文主体

**节奏要求：**
- 短句为主，一句话一行
- 每句不超过25字
- 每3-4句留一段空行
- 禁止：长复合句、学术解释、空洞情绪词

**必备元素（至少含2个）：**
- 具体场景（深夜/宿舍/午饭时）
- 人物反应（沉默/秒读/已读不回）
- 对话细节（"难怪我每次生气他都跑"）

### 2.4 互动引导（结尾）

**必须用问句，引发评论：**

| 类型 | 示例 |
|------|------|
| 求共鸣 | "你们有没有被AI说中的经历？" |
| 求意见 | "你们觉得AI说得准吗？" |
| 求分享 | "你们是什么类型？评论区告诉我" |

## 四、去AI味技巧

### 4.1 核心原则
- 不要说"测试显示"、"AI分析"
- 用"它说"、"发给它看了"代替
- 保留口语化和不完整句式

### 4.2 增加真实细节

| 元素 | 示例 |
|------|------|
| 时间 | "凌晨两点"、"冷战了3天" |
| 地点 | "在宿舍"、"她秒读已读不回" |
| 人物 | "我妈说"、"我室友说" |
| 反应 | "沉默了很久"、"愣了一下" |

### 4.3 词汇替换

| AI味词汇 | 人话词汇 |
|-----------|----------|
| "测试结果显示" | "它说" |
| "具有较高准确性" | "还挺准的" |
| "分析如下" | 不需要直接说分析 |
| "非常震惊" | "我愣住了😱" |
| "大家有什么看法" | "你们觉得呢？" |

## Telegram 消息接收问题排查

### 问题现象
- Typhoon 在 Telegram 上给 @CloudMap243bot 发消息，我没有回应
- 但 Telegram 状态显示 ON · OK，会话 `agent:main:telegram:direct:8755896074` 为 running

### 排查过程
1. 检查 Telegram 配置：`dmPolicy: "pairing"`（DM 需要配对）
2. Telegram session 存在且 running，但 history 里只有 1 条 assistant 消息，无用户消息
3. `sessions_send` 发送测试消息超时 → 会话处于假死状态
4. 配对列表：`No pending telegram pairing requests`

### 结论与建议
- **根因**：`dmPolicy: "pairing"` 配对状态可能失效或过期
- **方案**：建议将 `dmPolicy` 改为 `allow`（完全开放），或重启 Gateway
- **待Typhoon确认是否执行**

### 代码改动建议
```bash
openclaw config set channels.telegram.dmPolicy allow
```

## 重要发现：飞书通知跨app ID问题

### 根因
飞书的 open_id 是**跟应用绑定的**，不同 app 的同一用户有不同的 open_id：
- `ou_06fa35c5e6f248f0196b6e0fd0e94c19` 是丰哥在 main app (cli_a930ccff237adbca) 的 ID
- notifications app (cli_a9438878acf8dbb4) 的 cron job 不能用这个 ID
- 需要丰哥先和 notifications bot 对话获取正确的 ID

### 解决方案
让丰哥在飞书里搜索 notifications bot（App ID: cli_a9438878acf8dbb4）并发送一条消息
